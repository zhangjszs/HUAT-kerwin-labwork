## 新建 ML 项目设计

**项目名**：基于 Titanic 数据集的生存预测分析
**存放位置**：`ml-practice/titanic-survival-prediction/`
**技术栈**：Python, Jupyter Notebook, pandas, sklearn, XGBoost, SHAP

### Notebook 结构

1. **问题定义** — 明确目标、评估指标选择理由（为什么用 F1 而不是 Accuracy）
2. **EDA** — 数据分布、缺失值分析、特征相关性可视化
3. **特征工程** — 缺失值填充策略、类别编码、新特征构造（如家庭规模）
4. **多模型对比** — 逻辑回归 / 随机森林 / XGBoost，统一评估框架
5. **模型解释** — 特征重要性分析（SHAP 或 feature_importances）
6. **结论与反思** — 模型局限性、改进方向

**关键原则**：每个步骤写清楚"为什么这样做"，不只是"做了什么"。
