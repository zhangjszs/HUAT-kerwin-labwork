# 迷宫寻路系统 (Maze Pathfinding System)

[![Qt Version](https://img.shields.io/badge/Qt-6.7+-green.svg)](https://www.qt.io/)
[![C++ Standard](https://img.shields.io/badge/C++-17-blue.svg)](https://en.cppreference.com/w/cpp/17)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

一个基于 Qt6 的迷宫寻路算法可视化系统，支持多种经典寻路算法的演示和对比。

## 功能特性

### 支持的算法
- **A* (A-Star)** - 启发式搜索算法，效率最高
- **BFS (广度优先搜索)** - 保证最短路径
- **DFS (深度优先搜索)** - 快速搜索但不一定最优
- **Dijkstra** - 单源最短路径算法
- **JPS (Jump Point Search)** - 针对网格优化的A*变种

### 核心功能
- 可视化展示算法搜索过程
- 支持 4方向/8方向 移动模式
- 多种启发式函数选择（曼哈顿/欧几里得/切比雪夫/Octile）
- 算法性能对比分析
- 迷宫生成（保证可解）
- 迷宫导入/导出
- 暗黑/明亮主题切换

## 算法说明

### A* 算法
- **时间复杂度**: O(E log V)
- **特点**: 使用启发式函数引导搜索，效率最高
- **适用场景**: 大多数寻路场景

### BFS 算法
- **时间复杂度**: O(V + E)
- **特点**: 保证找到最短路径（无权图）
- **适用场景**: 需要最短路径且不考虑权重

### DFS 算法
- **时间复杂度**: O(V + E)
- **特点**: 快速搜索，但不一定找到最短路径
- **适用场景**: 快速探索，路径优化不重要

### Dijkstra 算法
- **时间复杂度**: O(E log V)
- **特点**: 保证最短路径（带权图）
- **适用场景**: 需要考虑不同地形代价

### JPS 算法
- **时间复杂度**: O(E log V)
- **特点**: 针对网格地图优化的A*变种，跳过无关节点
- **适用场景**: 大规模网格地图

## 设计思考

### 为什么选择 A* 作为默认算法？

BFS 保证最短路径但搜索节点多；DFS 快但不最优；Dijkstra 适合带权图但在等权网格上效率低于 A*。A* 通过启发函数将搜索方向引导向终点，在保证最优解的前提下大幅减少搜索节点数。

### JPS 的核心优化原理

JPS 利用网格地图的对称性，跳过不可能成为最优路径一部分的中间节点，直接跳到“跳点”（方向改变的关键节点）。在开阔地图上可将搜索节点减少60-80%。

### 启发函数的选择

- **曼哈顿距离**：4方向移动时最优（不高估实际代价，保证 admissibility）
- **Octile距离**：8方向移动时最优
- **欧几里得距离**：可能高估，破坏 admissibility，导致非最优路径

## 常见问题

### Q: 为什么有时生成的迷宫看起来很简单？
A: 系统使用保证可解的迷宫生成算法，高障碍物密度可能导致简单路径。尝试调整密度滑块。

### Q: JPS算法为什么比A*快？
A: JPS通过跳过中间节点减少搜索空间，特别适合开阔地图。但在复杂迷宫中优势不明显。

### Q: 如何保存当前迷宫？
A: 点击"保存"按钮或按 `Ctrl+S`，迷宫将保存为文本文件。

### Q: 支持多大的迷宫？
A: 系统支持 5x5 到 100x100 的迷宫大小。推荐 20x20 到 50x50 获得最佳体验。
